# Contributing to Pixelyf

First off, thank you for considering contributing to Pixelyf! It's people like you that make Pixelyf such a great tool.

Welcome to our community. We welcome contributions from everyone.

---

## Code of Conduct

By participating in this project, you agree to abide by our standards of respect, inclusivity, and collaboration. Please be helpful, friendly, and respectful in all interactions.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please create a GitHub Issue. 
- Use a clear and descriptive title.
- Describe the exact steps which reproduce the problem.
- Provide information about your environment (OS, Browser, Node.js version).

### Suggesting Enhancements
If you have an idea to improve Pixelyf, feel free to open an issue suggesting the enhancement.
- Explain the behavior you would like to see and why it would be useful.
- Provide mockups or flow diagrams if applicable.

### Pull Requests
We accept Pull Requests (PRs) for bug fixes and new features. To make sure your PR gets merged:
1. **Fork the Repository** and create your branch from `main`.
2. **Setup your environment** by copying `.env.example` to `.env` and installing dependencies:
   ```bash
   npm install
   ```
3. **Write clean code** and adhere to the project's formatting guidelines.
4. **Test your changes**: Run `npm run build` to ensure the project compiles successfully.
5. **Open a Pull Request** with a clear description of the changes and link the related issue.
