# Pixelyf에 기여하기

우선 Pixelyf에 기여해 주시려는 생각을 해주셔서 감사합니다! 여러분과 같은 분들 덕분에 Pixelyf가 이렇게 훌륭한 도구가 될 수 있습니다.

저희 커뮤니티에 오신 것을 환영합니다. 모든 분들의 참여를 환영합니다.

---

## 행동 강령

본 프로젝트에 참여함으로써 귀하는 존중, 포용성 및 협력이라는 당사의 기준을 준수하는 데 동의하는 것입니다. 모든 상호 작용에서 도움을 주고 친절하며 존중하는 태도를 보여주시기 바랍니다.

## 어떻게 기여할 수 있나요?

### 버그 신고

버그를 발견하시면 GitHub 이슈를 생성해 주세요.

- 명확하고 설명적인 제목을 사용하세요.
- 문제를 재현하는 정확한 단계를 설명하세요.
- 사용 환경 정보(운영체제, 브라우저, Node.js 버전)를 제공해 주세요.

### 기능 개선 제안

Pixelyf를 개선할 아이디어가 있다면 언제든지 개선 사항을 제안하는 이슈를 열어주세요.

- 원하는 동작 방식과 그 동작 방식이 유용한 이유를 설명해 주세요.
- 가능하다면 모형이나 흐름도를 제공해 주세요.

### 풀 리퀘스트

저희는 버그 수정 및 새로운 기능 구현을 위한 풀 리퀘스트(PR)를 환영합니다. PR이 병합되도록 하려면 다음 사항을 확인하세요.

1. **저장소를 포크하고** main 에서 브랜치를 생성하세요.
2. **설치 환경을 설정** 하려면 .env.example 을 복사해서 .env 파일을 만들고 필요한 파일을 설치하세요.
   ```bash
   npm install
   ```
3. **깔끔한 코드를 작성** and adhere to the project's formatting guidelines.
4. **변경 사항을 테스트하세요**: Run `npm run build` to ensure the project compiles successfully.
5. 변경 사항에 대한 명확한 설명과 함께 **풀 리퀘스트를 열고 관련 이슈를 링크하세요.**
